PROJECT OVERVIEW
================================================================================

FastMCP-x is a full-stack document-aware query assistant that combines a Model 
Context Protocol (MCP) server backend with a modern Next.js frontend. It enables 
users to ingest documents, perform semantic searches, query structured data, and 
interact with an AI assistant through a professional web interface.

Core Purpose: Provide an intelligent document management and querying system 
with AI-powered insights, user authentication, and a responsive UI.

DATA FLOW
================================================================================

1. DOCUMENT INGESTION
   â”€â”€â”€â”€â”€â”€â”€â”€â”€â”€â”€â”€â”€â”€â”€â”€â”€â”€
   1. User uploads file via vault UI
   2. File sent to /api/ingest endpoint
   3. Bridge server invokes ingest_file_tool
   4. FastMCP server:
      â€¢ Copies file to storage/ directory
      â€¢ Extracts text using appropriate parser
      â€¢ Stores as dict: {content, filename, filepath}
      â€¢ Returns success/error
   5. Frontend displays confirmation


2. QUERY PROCESSING
   â”€â”€â”€â”€â”€â”€â”€â”€â”€â”€â”€â”€â”€â”€â”€â”€
   1. User types message in chat
   2. Frontend sends query to /api/chat/query API route
   3. API route forwards to bridge server /api/query
   4. Bridge server calls answer_query_tool with context
   5. FastMCP server:
      â€¢ Performs semantic search on documents
      â€¢ Chunks documents (600 chars, 50 char overlap)
      â€¢ Sends relevant chunks + query to LLM
      â€¢ Ollama generates response
      â€¢ Returns response to frontend
   6. Frontend displays message and updates history


3. CONVERSATION CONTEXT
   â”€â”€â”€â”€â”€â”€â”€â”€â”€â”€â”€â”€â”€â”€â”€â”€â”€â”€â”€â”€
   â€¢ Frontend maintains last 10 messages in localStorage
   â€¢ Sends last 6-10 messages with each query
   â€¢ Backend uses context to resolve pronouns/references
   â€¢ Improves response accuracy for follow-up questions


4. WEB SEARCH
   â”€â”€â”€â”€â”€â”€â”€â”€â”€â”€
   1. User query triggers web search tool
   2. Tavily API fetches top result
   3. BeautifulSoup4 extracts content
   4. Content passed to LLM for summarization
   5. Summarized response returned to user

TROUBLESHOOTING COMMON ISSUES
================================================================================

Ollama not found:
  Install from https://ollama.ai and ensure service is running

Documents not loading:
  Verify storage/ directory exists and has read permissions

Auth redirect fails:
  Add http://localhost:3000/auth/callback to Supabase redirect URLs

Session not persisting:
  Clear browser cookies, check middleware configuration

Port in use:
  Change port with: npm run dev -- -p 3001

Module not found:
  Run: npm install (in frontend directory)

RLS policy errors:
  Verify user is authenticated, check Supabase policies

Sidebar state not saving:
  Check localStorage is enabled in browser
